# Настройка сцены Blast — пошагово для Cocos Creator

**Разделение логики и отображения:** логика в `Game/` (GameBoard, GameState, GameConfig), отображение в `View/` (BoardView, TileView, GameUI); BlastGame только связывает их и обрабатывает ввод.

Ниже всё привязано к **вашей** структуре: Canvas → Background, Score, GameDesk.

---

## Шаг 1. Компонент BlastGame на Canvas

1. В **Node Tree** (дерево узлов слева) кликните по **Canvas**.
2. Внизу справа откроется **Inspector** (свойства узла).
3. Нажмите **Add Component**.
4. В поиске введите `BlastGame` и выберите скрипт **BlastGame**.
5. Компонент появится в списке компонентов Canvas. Поля **Board View** и **Game UI** пока оставьте пустыми — заполним в шагах 3 и 5.

---

## Шаг 2. Компонент BoardView на GameDesk

Игровое поле (сетка тайлов) строится **внутри** узла GameDesk. Спрайты разделены по категориям.

1. В Node Tree выберите **GameDesk**.
2. В Inspector нажмите **Add Component** → найдите **BoardView** → добавьте.
3. В компоненте BoardView настройте:

   **Размеры:** **Tile Size** (например 64), **Spacing** (например 4). **Tile Prefab** — префаб обычного тайла. **Bomb Tile Prefab** — по желанию отдельный префаб для бомб (ракета Г/В, бомба, бомба-макс); если задан, для ячеек со значениями 5–8 создаётся он вместо Tile Prefab (логика клика по бомбам — в игре, префаб только для отображения).

   **Idle-анимация бомб:** для тайлов «бомба», «горизонтальная бомба», «вертикальная бомба», «максимальная бомба» можно настроить пульс (увеличение раз в N секунд): **Bomb Idle Interval** — интервал в секундах (по умолчанию 5); **Bomb Idle Anim Duration** — длительность одного пульса в секундах (по умолчанию 0.35). На сцене ничего дополнительно ставить не нужно — анимация запускается сама для этих тайлов. Если используете префаб тайла, компонент **TileView** должен висеть на том узле, который нужно масштабировать (обычно корень префаба).

   **Категория «Обычные»** — **Ordinary Frames**: массив из **5** спрайтов по цветам (0 красный, 1 зелёный, 2 синий, 3 жёлтый, 4 сиреневый). Можно использовать текстуры `block_red`, `block_green`, `block_blue`, `block_yellow` и один для сиреневого.

   **Категория «Бомба»** — **Bomb Frame**: один спрайт для обычной бомбы (радиус 3×3), например `block_bomb`.

   **Категория «Бомба горизонтальная»** — **Bomb Horizontal Frame**: спрайт для уничтожения всей строки, например `block_rockets_horisontal`.

   **Категория «Бомба вертикальная»** — **Bomb Vertical Frame**: спрайт для уничтожения всего столбца, например `block_rakets` (вертикальная ракета).

   **Остальное:** **Tile Sprite Frame** — общий fallback; **Tile Frames** — массив для бомбы-макс, «очистить всё», телепорт (индексы 8–10) и запас для обычных/бомб, если категории не заполнены.

**Механика:** обычные тайлы — клик по группе 2+ одного цвета сжигает только эту группу и даёт спецтайл при 5–9/10+. Бомбы (5–8): клик по бомбе сжигает тайлы в радиусе R в этой клетке; если прилегает группа бомб — сжигаются тайлы в объединении радиусов R; затем цепная реакция. Очистить всё (9) и телепорт (10) — как раньше.

## Шаг 3. Связать GameDesk с BlastGame

1. Снова выберите **Canvas** в Node Tree.
2. В Inspector найдите компонент **BlastGame**.
3. В поле **Board View** перетащите из Node Tree узел **GameDesk** (тот, на котором висит BoardView).

Так BlastGame узнает, где рисовать тайлы.

---

## Шаг 4. Компонент GameUI на Score

У вас уже есть узел **Score** с дочерними **Level** и **ScoreNumber**. Используем их для ходов и очков.

1. В Node Tree выберите узел **Score** (родитель Level и ScoreNumber).
2. В Inspector нажмите **Add Component** → найдите **GameUI** → добавьте.
3. В компоненте GameUI нужно заполнить слоты (перетаскиванием из Node Tree):
   - **Score Label** — узел с очками (например **ScoreNumber**).
   - **Moves Label** — узел с ходами (например **Level**).
   - **Bomb Booster Count Label** — обязательно узел **booster count**, который является дочерним у **Booster Bomb** (именно он, не от Teleport). По умолчанию показывает 5.
   - **Teleport Booster Count Label** — обязательно узел **booster count**, который является дочерним у **Booster Teleport** (именно он, не от Bomb). По умолчанию показывает 3. Если оба слота указывают на один и тот же Label, счётчики будут работать неправильно.
   - **Win Panel** / **Lose Panel** — узлы с надписями «Победа!» и «Проигрыш» (по умолчанию неактивны).

---

## Шаг 5. Связать Score и бустеры с BlastGame

1. Выберите **Canvas** в Node Tree.
2. В компоненте **BlastGame** перетащите:
   - **Game UI** — узел **Score** (на котором висит GameUI).
   - **Booster Bomb Btn** — узел **Booster Bomb** (кнопка: при клике списывается одно использование и включается режим «бомба»; следующий клик по полю — сжигание тайлов в радиусе R клеток).
   - **Booster Teleport Btn** — узел **Booster Teleport** (кнопка: при клике списывается одно использование и включается режим «телепорт»; два клика по полю — обмен двух тайлов местами).

Бустеры под заголовком «Бустеры» и спецтайлы на поле (ракеты, бомбы, телепорт) работают **отдельно**: счётчики кнопок не связаны с бустерами, появляющимися при сборе групп 5–9 или 10+ тайлов.

## Шаг 6. Проверка

1. Сохраните сцену (Ctrl+S).
2. Нажмите **Play** (треугольник вверху) или **Preview**.
3. Должно появиться поле тайлов внутри GameDesk; клик по тайлу сжигает группу, тайлы падают, очки и ходы обновляются в Score/Level и ScoreNumber.

## Тайлы по категориям и как это работает

### Категории тайлов

| Категория | ID | Описание |
|-----------|----|----------|
| **Обычные (по цветам)** | 0–4 | 5 цветов: красный, зелёный, синий, жёлтый, сиреневый. Связная группа одного цвета сжигается по клику (от 2 тайлов). |
| **Ракета горизонтальная** | 5 | При клике — **только очищение строки** (горизонтальный ряд). Если в ряду попадается бомба — она срабатывает по своей логике (цепная реакция). |
| **Ракета вертикальная** | 6 | При клике — **только очищение столбца** (вертикальный ряд). Попавшие в столбец бомбы срабатывают по своей логике (цепная реакция). |
| **Обычная бомба (3×3)** | 7 | Радиус R (3×3) от клетки; если группа прилегающих бомб (5–8) — объединение эффектов (ракета → ряд/столбец, бомба → радиус). Цепная реакция. |
| **Бомба-макс (5×5)** | 8 | Радиус 5×5 от клетки; в группе бомб — объединение эффектов по типу каждой. Цепная реакция. |
| **Очистить всё** | 9 | При активации сжигает **всё поле**. |
| **Телепорт** | 10 | Не сжигает: по первому клику — выбор тайла, по второму — **обмен двух тайлов** местами (тратится ход). |

Обычные тайлы (0–4) создают спецтайлы при сборе группы: **5–9** тайлов → случайный бустер (ракета Г/В, бомба, телепорт); **10+** → мощный бустер (бомба-макс или «очистить всё») + бонусные ходы.

### Как работает ход в Blast

1. **Клик по обычному тайлу (цвет 0–4)**  
   Считается связная группа одного цвета. Если в группе **меньше 2** — только подсветка (пульс), ход не тратится. Если **2 и больше** — сжигается **только эта группа** (без радиуса R). При 5–9/10+ в центр ставится спецтайл и при 10+ даются бонусные ходы. Обычные тайлы не трогают радиус R.

2. **Клик по ракете (горизонтальная 5 / вертикальная 6)**  
   Очищается **только** строка (горизонтальная ракета) или **только** столбец (вертикальная). Если в этом ряду оказывается другая бомба — она срабатывает по своей логике (цепная реакция). Ход тратится.

3. **Клик по бомбе (7) или бомбе-макс (8)**  
   Одна клетка — сжигаются тайлы в радиусе R (3×3 или 5×5). Если есть группа прилегающих бомб (5–8) — объединение эффектов: ракета даёт свой ряд/столбец, бомба — свой радиус. Затем цепная реакция. Ход тратится.

4. **Клик по «очистить всё» (9)**  
   Зона эффекта — всё поле, затем цепная реакция. Ход тратится.

5. **Телепорт (на поле или кнопка «Бустер телепорт»)**  
   Два клика по полю — обмен двух тайлов. Кнопка телепорта списывает свой счётчик; телепорт-тайл на поле тратит ход.

6. **Кнопка «Бустер бомба»**  
   Клик по кнопке списывает использование; следующий клик по полю — сжигание **радиус R** в этой клетке с цепной реакцией.

После любого сжигания: гравитация (тайлы падают), дозаполнение сверху, проверка победы/поражения и при отсутствии ходов — до 3 перемешиваний или проигрыш.

### Справка: классическая механика (своп + матч 3+)

В описанном в примечаниях варианте: клик по тайлу выделяет его, клик по соседу меняет их местами; при матче 3+ тайлы сгорают, иначе — возврат. Спецбомбы создаются от 4+ в ряд (L/T — обычная бомба 3×3, горизонтальный ряд — ракета по строке, вертикальный — по столбцу). Цепная реакция: взрыв одной бомбы добавляет задетые бомбы в очередь и обрабатывается через очередь до конца. Комбо: Г+Г — три ряда, В+В — три столбца, Г+В — крест (ряд + столбец); 3×3 + линейная — оба эффекта по очереди.





При клике игрока на обычный тайл происходит следующее:

Определение группы – система находит все смежные (по горизонтали и вертикали) тайлы того же цвета/типа, что и кликнутый. Обычно группа должна состоять минимум из двух (или трёх, в зависимости от правил) элементов, чтобы её можно было удалить.

Удаление – вся найденная группа исчезает с поля. Это может сопровождаться анимацией взрыва или исчезновения.

Падение (гравитация) – оставшиеся над пустотами тайлы падают вниз (или заполняются сверху новыми).

Заполнение новыми тайлами – на освободившиеся верхние позиции генерируются новые случайные тайлы.

Проверка новых совпадений – после падения система проверяет, не образовались ли новые группы одинаковых тайлов. Если да – они автоматически удаляются (комбо), и процесс повторяется, пока есть совпадения.

2. Бомбы – специальные тайлы
Бомбы появляются на поле как результат специальных комбинаций (например, при совпадении 4 или 5 тайлов). Они обладают особыми эффектами при активации.

2.1. Обычная бомба (радиусная)
Внешний вид: часто выделяется значком бомбы или звёздочкой.

Действие: при активации взрывает все тайлы в квадрате 3×3 (или ином радиусе) с центром на себе.

Особенности: уничтожает также другие бомбы, попавшие в зону поражения, что может запустить цепную реакцию.

2.2. Горизонтальная бомба
Внешний вид: может иметь иконку с горизонтальными полосками или стрелками.

Действие: уничтожает все тайлы в своей строке (ряду) по горизонтали.

Особенности: взрыв проходит через всю ширину поля, не ограничиваясь ближайшими соседями. Если в строке встречаются другие бомбы, они активируются.

2.3. Вертикальная бомба
Внешний вид: иконка с вертикальными полосками.

Действие: уничтожает все тайлы в своём столбце по вертикали.

Особенности: аналогично горизонтальной, взрыв распространяется на весь столбец, активируя другие бомбы на своём пути.

3. Активация бомб
Бомба может быть активирована двумя способами:

Прямой клик игрока – если игрок кликает на бомбу, она срабатывает немедленно, даже если рядом нет группы одинаковых тайлов. Иногда бомбу можно активировать только в составе группы (например, если она входит в группу одинаковых тайлов при клике на обычный тайл).

Косвенная активация (через взрыв другой бомбы) – если бомба попадает в зону действия другой бомбы (радиус, строка или столбец), она тоже взрывается, но не мгновенно, а ставится в очередь на последующее срабатывание. Это основа цепной реакции.

4. Цепная реакция (каскад взрывов)
Цепная реакция возникает, когда взрыв одной бомбы задевает другие бомбы, которые затем тоже взрываются, и процесс может повторяться многократно.

Алгоритм цепной реакции:

Начальный взрыв – происходит по клику игрока (на группу обычных тайлов или на бомбу). Удаляются соответствующие тайлы, и все бомбы, которые должны сработать на этом этапе, помечаются как активированные.

Обработка очереди – система поочерёдно обрабатывает каждую активированную бомбу:

При взрыве бомбы удаляются все тайлы в её зоне действия.

Если среди удаляемых тайлов есть другие бомбы, они не взрываются сразу, а добавляются в конец очереди (чтобы избежать бесконечного цикла и сохранить порядок).

Повтор – очередь обрабатывается, пока все активированные бомбы не сработают.

Падение и новые совпадения – после завершения всех взрывов применяется гравитация: тайлы падают вниз, а сверху добавляются новые. Затем проверяются естественные совпадения (группы из 3+ одинаковых тайлов). Если они есть, они удаляются, и при необходимости снова активируются бомбы, которые могли образоваться или попасть в эти группы. Этот процесс может создать дополнительные цепочки.

Завершение хода – когда больше нет ни взрывов бомб, ни естественных совпадений, ход считается оконченным, и игрок может делать следующий клик.

Важные нюансы:

Если одновременно активируются несколько бомб, порядок их обработки может влиять на результат. Обычно используется FIFO (очередь) или случайный порядок, но важно, чтобы каждая бомба сработала ровно один раз.

Горизонтальные и вертикальные бомбы, пересекаясь, могут создавать мощные комбинации (например, если горизонтальная и вертикальная бомбы активируются в одной клетке, они уничтожат и строку, и столбец, а также все бомбы на их пересечении).

При падении новые бомбы, упавшие на место сработавших, не активируются автоматически – они становятся обычными специальными тайлами, которые можно активить позже.

5. Пример для наглядности
Представим поле 5×5. Есть:

Обычная бомба в центре (координаты 3,3).

Горизонтальная бомба в строке 2.

Вертикальная бомба в столбце 4.

Игрок кликает на обычный тайл рядом с обычной бомбой, и группа удаляется, но бомба не была затронута. Затем игрок кликает на обычную бомбу. Она взрывается, уничтожая область 3×3 вокруг (клетки 2,2 – 4,4). В эту область попадает горизонтальная бомба (2,4) и вертикальная бомба (4,4). Они добавляются в очередь.

Обработка очереди:

Сначала срабатывает горизонтальная бомба (2,4): уничтожает всю строку 2. На пути могут быть другие бомбы – допустим, в строке 2 есть ещё одна обычная бомба в клетке (2,1). Она добавляется в очередь.

Затем срабатывает вертикальная бомба (4,4): уничтожает столбец 4. При этом могут быть затронуты другие бомбы – например, в столбце 4 есть ещё одна горизонтальная бомба в клетке (1,4). Она добавляется в очередь.

Далее срабатывает обычная бомба из (2,1) – взрывает свой радиус, возможно задевая новые бомбы.

И так далее, пока очередь не опустеет.

После всех взрывов тайлы падают, и система проверяет новые совпадения. Если при падении образовалась группа из трёх одинаковых тайлов, они удаляются, и возможно появление новых бомб или активация уже существующих.

6. Рекомендации по реализации
Система очереди – обязательна для корректной цепной реакции.

Защита от зацикливания – каждая бомба должна срабатывать только один раз за ход (можно помечать её как "уже использована").

Анимации – для лучшего восприятия стоит показывать взрывы последовательно, а не одновременно, чтобы игрок видел цепочку.

Комбо-множители – можно начислять дополнительные очки за каждый шаг цепной реакции.

Такая механика создаёт глубокий и увлекательный геймплей, где игрок может стратегически размещать бомбы и провоцировать мощные каскады.
